🤖 What is Nexus AI Agent?

Nexus AI Agent is a sophisticated multi-agent orchestration platform designed to handle complex workflows and retrieval-augmented generation (RAG) tasks. Think of it as having a team of AI specialists working together to answer your questions.

The system uses three specialized AI agents:

  • 🎯 Supervisor: The coordinator that decides which agent should handle your request
  • 🔍 Researcher: Searches through your uploaded documents to find relevant information
  • ✍️ Writer: Generates comprehensive, well-written responses based on the research
💡 Why is this useful?

Instead of manually searching through documents and writing summaries, you can ask questions in natural language and get accurate, context-aware answers instantly. The AI agents work together to provide comprehensive responses based on your actual documents.

✨ Key Features
🤖 Multi-Agent Orchestration
Three specialized AI agents work together: Supervisor coordinates, Researcher finds information, Writer creates responses.
📄 Document Upload
Upload PDF documents and the system will automatically process, index, and make them searchable.
🔍 Semantic Search
Uses advanced embeddings to understand the meaning of your questions and find relevant information.
💬 Interactive Chat
Beautiful, modern chat interface with real-time responses and agent status visualization.
🏠 Local AI
Uses Ollama with Llama 3 for completely local, private AI processing - no data leaves your machine.
🎨 Modern UI
Dark mode interface built with React, TailwindCSS, and smooth animations.
🛠️ Technology Stack

Backend Technologies

Python 3.10+
The programming language used for the backend. Known for its simplicity and extensive AI/ML libraries.
FastAPI
A modern, fast web framework for building APIs. Provides automatic documentation and high performance.
LangGraph
Framework for building stateful, multi-agent applications. Manages the workflow between agents.

Framework for developing applications powered by language models. Provides tools for RAG and document processing.

Vector database for storing and searching document embeddings. Enables fast semantic search.
Ollama
Runs large language models locally. Uses Llama 3 for AI reasoning and response generation.

Frontend Technologies

TypeScript
Adds type safety to JavaScript. Helps catch errors early and improves code quality.
Vite
Next-generation frontend tooling. Provides fast development server and optimized builds.
TailwindCSS
Utility-first CSS framework. Makes it easy to create beautiful, responsive designs.

  • Node.js 18 or higher
    Download from nodejs.org. This includes npm (Node Package Manager).
  • Git
    Download from git-scm.com. Used for cloning the repository.
  • Ollama
    Download from ollama.com. Runs AI models locally on your machine.
⚠️ Important

Make sure Ollama is installed and running before starting the application. The system uses Ollama to run the Llama 3 AI model locally.

📥 Installation Guide

Follow these steps to install Nexus AI Agent on your computer:

1
Clone the Repository

Open a terminal or command prompt and run:

git clone 
cd Nexus-AI-Agent
2
Set Up Ollama

First, pull the Llama 3 model:

ollama pull llama3

Then start the Ollama server:

ollama serve
💡 Note

Keep the Ollama server running in a separate terminal window. The system needs it to process AI requests.

3
Set Up Backend

Navigate to the backend directory and create a virtual environment:

cd backend

# Create virtual environment
python -m venv .venv

# Activate virtual environment
# On Windows:
.venv\Scripts\activate

# On Mac/Linux:
source .venv/bin/activate

Install Python dependencies:

pip install -r requirements.txt
💡 What is a virtual environment?

A virtual environment is an isolated Python environment that keeps project dependencies separate from your system Python. This prevents conflicts between different projects.

4
Set Up Frontend

Open a new terminal window and navigate to the frontend directory:

cd frontend

# Install Node.js dependencies
npm install

This will download all required JavaScript packages.

✅ Installation Complete!

You've successfully installed Nexus AI Agent. Now let's learn how to run it locally.

🚀 Running Locally

To run Nexus AI Agent on your local machine, you need to start three services:

1
Start Ollama Server

In a terminal window, run:

ollama serve

Keep this terminal open. Ollama needs to keep running to process AI requests.

2
Start Backend Server

In a new terminal window, navigate to the backend directory and activate the virtual environment:

cd backend

# Activate virtual environment (if not already activated)
# Windows:
.venv\Scripts\activate

# Mac/Linux:
source .venv/bin/activate

# Start the FastAPI server
uvicorn main:app --reload

You should see output indicating the server is running on http://localhost:8000

3
Start Frontend Server

In another new terminal window, navigate to the frontend directory:

cd frontend

# Start the development server
npm run dev

You should see output indicating the server is running on http://localhost:5173

🎉 Application is Running!

Open your web browser and go to http://localhost:5173 to use Nexus AI Agent!

💡 Quick Reference

Ollama: ollama serve
Backend: uvicorn main:app --reload (in backend directory)
Frontend: npm run dev (in frontend directory)
Access: http://localhost:5173

💬 Using the Application

Uploading Documents

Before asking questions, you should upload some PDF documents for the AI to reference:

  1. Click on the "Knowledge Base" tab in the navigation bar
  2. Drag and drop a PDF file onto the upload area, or click to browse
  3. Wait for the upload to complete (you'll see a success message)
  4. The document is now indexed and ready for questions!

Asking Questions

Once you've uploaded documents, you can start asking questions:

  1. Click on the "Mission Control" tab
  2. Type your question in the input field at the bottom
  3. Press Enter or click the send button
  4. Watch as the AI agents work together to answer your question
💡 Tips for Better Results
  • Be specific in your questions
  • Upload multiple related documents for comprehensive answers
  • The system works best with text-based PDFs (not scanned images)
  • Watch the "Live Agent State" panel to see which agent is working
🌐 Deployment Guide

Want to share your Nexus AI Agent with others? Here's how to deploy it:

🖥️ Option 1: Local Network

Best for: Sharing with people on the same network

Steps:

  1. Find your computer's IP address
  2. Start backend with: uvicorn main:app --host 0.0.0.0 --port 8000
  3. Start frontend with: npm run dev -- --host
  4. Share http://YOUR_IP:5173 with others
☁️ Option 2: Cloud Deployment

Best for: Public access from anywhere

Platforms:

  • Railway: Easy Python deployment
  • Render: Free tier available
  • AWS/Google Cloud: Scalable infrastructure
  • DigitalOcean: Simple VPS hosting
🐳 Option 3: Docker

Best for: Consistent deployment across environments

Steps:

  1. Create Dockerfiles for backend and frontend
  2. Build images: docker-compose build
  3. Run: docker-compose up
⚠️ Deployment Considerations
  • Ollama requires significant RAM (8GB+ recommended)
  • Consider using a GPU for better AI performance
  • Set up proper authentication for public deployments
  • Configure CORS settings for production
🔧 Troubleshooting
❌ "Failed to fetch" error in the chat

This means the frontend can't connect to the backend. Check:

  • Is the backend server running? (You should see "Uvicorn running on http://localhost:8000")
  • Is the backend running on port 8000?
  • Are there any error messages in the backend terminal?
❌ Ollama connection error

If you see errors about Ollama:

  • Make sure Ollama is installed: ollama --version
  • Start Ollama server: ollama serve
  • Pull the Llama 3 model: ollama pull llama3
  • Check if Ollama is running on port 11434
❌ Python module not found errors

If you see "ModuleNotFoundError":

  • Make sure you activated the virtual environment
  • Reinstall dependencies: pip install -r requirements.txt
  • Check Python version: python --version (should be 3.10+)
❌ npm install fails

If npm install fails:

  • Check Node.js version: node --version (should be 18+)
  • Clear npm cache: npm cache clean --force
  • Delete node_modules folder and try again
  • Try using a different terminal or restart your computer
❌ Port already in use

If you see "Address already in use":

  • Another application is using that port
  • Find and close the application using the port
  • Or use a different port: uvicorn main:app --port 8001
❓ Frequently Asked Questions
Q: Is my data sent to the cloud?

No! Nexus AI Agent runs completely locally. All AI processing happens on your machine using Ollama. Your documents and conversations never leave your computer.

Q: What types of documents can I upload?

Currently, the system supports PDF documents. The PDFs should contain text (not just scanned images) for best results.

Q: How much RAM do I need?

We recommend at least 8GB of RAM. The Llama 3 model requires significant memory to run efficiently.

Q: Can I use a different AI model?

Yes! You can modify the model in backend/graph/nodes.py. Change ChatOllama(model="llama3") to any model available in Ollama.

Q: How do I reset the vector database?

Delete the backend/chroma_db folder. The system will create a new one when you upload documents.

Q: Can I use OpenAI instead of Ollama?

Yes! Modify backend/graph/nodes.py to use ChatOpenAI instead of ChatOllama. You'll need to set your OpenAI API key in a .env file.
